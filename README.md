# Очередь отложенных вызовов (Deferred Function Queue)

## Описание проекта

В рамках данного проекта реализована очередь отложенных вызовов функций и методов классов на языке C++.

Проект основан на предметной области «Школа», разработанной в предыдущих семинарах. В нём используются классы Student, Teacher и School.

Основная идея проекта заключается в том, чтобы можно было откладывать выполнение функций и методов, помещая их в очередь, а затем выполнять их в нужный момент времени.

---

## Цель работы

Целью работы является реализация механизма отложенного выполнения функций с использованием современных возможностей языка C++:

- вариадических шаблонов;
- стандартных контейнеров;
- кортежей (std::tuple);
- механизма std::apply;
- полиморфизма.

---

## Постановка задачи

Необходимо реализовать объект — очередь отложенных вызовов, которая должна поддерживать:

- enqueue — добавление отложенного вызова функции или метода;
- run — выполнение одного отложенного вызова;
- run_all — выполнение всех отложенных вызовов.

Особенность задачи заключается в том, что функции могут иметь произвольное количество аргументов разных типов, поэтому требуется использовать вариадические шаблоны.

---

## Структура решения

Решение состоит из трёх основных компонентов:

1. Интерфейс отложенного вызова — задаёт общий метод выполнения задачи.
2. Шаблонный класс отложенного вызова — хранит функцию и её аргументы.
3. Очередь отложенных вызовов — управляет хранением и выполнением задач.

Каждый отложенный вызов представляется как объект, который можно выполнить через единый интерфейс.

---

## Принцип работы

1. В очередь добавляются задачи (функции или методы с аргументами).
2. На этапе добавления задачи не выполняются.
3. Метод run выполняет одну задачу из очереди.
4. Метод run_all выполняет все задачи по порядку.

Очередь работает по принципу FIFO (первый добавлен — первый выполнен).

---

## Использование предметной области

В проекте используется модель «Школа», включающая:

- учеников (Student);
- учителей (Teacher);
- школу (School).

Через очередь выполняются следующие действия:

- назначение классного руководителя;
- добавление оценок;
- вывод дневника ученика;
- вывод информации об учителе;
- добавление объектов в школу;
- вывод списков учеников и учителей.

---

## Демонстрация работы

Программа сначала добавляет задачи в очередь, после чего выполняет:

- один отложенный вызов через метод run;
- остальные вызовы через метод run_all.

Вывод программы показывает последовательное выполнение всех задач, а также подтверждает корректную работу очереди.
